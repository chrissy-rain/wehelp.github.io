* 軟體技術日新月異，如何確定選擇投入的領域是正確有回報的？

  1. 軟體技術日新月異但就目前的情況來說還沒有辦法完全取代「人」，所以目前的投入是可以有回報的。
  2. 軟體技術進步會留下更好用的方法或手段，如果能學會這些新技術這會是一個很好的機會。
  3. 回報的多寡跟投入的精力有很大的關係，想要收穫多一點就要比別人努力一點，只要是能讓自己有所成長那就是正確的。
  4. 回報可分成抽象跟具體的兩部分
    • 抽象：知識、技能的成長，在學習過程中解決問題的成就感。
    • 具體：跨足新領域的薪水可能一開始比不上原有的薪水，但是未來可期。
    5. 對於軟體領域這塊我並不熟悉，我能想到的方法是先打好基礎（例程式怎麼寫得邏輯），其他細項則是透過人力銀行查看目前市場需要的職缺內容及其要求的技能，再補足缺漏的部分。

* 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

事件：因為朋友借錢不還，產生被欺騙的負面情緒

處理方法:

  1. 積極聯絡朋友,督促他要還錢,並給予一個期限
  2. 時間到卻未還款,視金額大小分不同的處理方法,小錢能承受這筆損失就當作拿錢看清一位朋友,以後再也不聯絡,大錢蒐集證據走法律途徑
  3. 事情告一段落就放下他,不要一直沉浸在負面情緒內
  4. 記取教訓,只要借錢就要先做好錢拿不回來的準備

* 關於這份申請網頁，分享一個開發時的技術心得。

第一次使用github發布網頁,雖然說是照著網頁上的提示一步步跟著做的,但還是會遇到不順利或是出現跟頁面寫得不一樣的狀況發生。

一開始會試著先自己摸索幾分鐘,發現還是不太懂後開始google.youtube上搜索答案,也就拼拼湊湊的完成了！不免俗的生出了一點點的成就感。

為了避免下次再出現這種狀況,我會把每一步都記錄下來,下次遇到狀況也比較好解決

如何在GitHub發布網頁

  1. 註冊GitHub帳號
  2. 準備要上傳的網頁資料夾（注意名稱大小寫）
  3. 開啟github →點選create repository
  4. Repository name要遵守規格填入：名稱.github.io
  5. 滑到網頁最底按點選create repository
  6. 選uploading an existing file
  7. 選（拖曳）資料夾到github
  8. commit changes 欄位輸入訊息 例如version1
  9. 選commit changes
  10. 在導覽列選settings →左側導覽列點 pages →(確定source在branch選項)branche改main →點save
  11. 重新點選pages →點visit site →出現404 File not found
  12. 因為上傳的是資料夾,所以在開啟的網址後加上/資料夾名稱/網頁名稱.html
更新網頁
  1. 返回到專案
  2. 選Add file →upload files
  3. 選（拖曳）資料夾到github
  4. commit chamges 輸入訊息 例如version2
